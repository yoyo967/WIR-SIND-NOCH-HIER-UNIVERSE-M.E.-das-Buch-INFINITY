## Prolog: Die ewige Wiederkehr

Nichts lernt einmalig. Jede Zivilisation, jede Wissenschaft, jede Weisheitstradition kennt den Zyklus: Handeln, Beobachten, Verstehen, Revidieren, erneut Handeln. Der Loop ist nicht das Zeichen eines unvollendeten Systems – er ist das Zeichen eines lebendigen.

## I. Definition: Was ist der Loop?

Der Loop ist der Feedback-Mechanismus, der die Chronik (Buch IV) zurück in das Wiki (Buch II) fließen lässt. Er ist die Brücke zwischen Vergangenheit (was geschah) und Gegenwart (was gilt).

**Der Loop besteht aus fünf Schritten:**

```
1. PROPOSE   → Agent schlägt Aktion/Änderung vor
2. IMPLEMENT → Aktion wird ausgeführt (im Test/Staging)
3. EXECUTE   → Ergebnis wird in Chronik aufgezeichnet
4. EVALUATE  → Ergebnis wird gegen Erwartung gemessen
5. COMMIT oder DISCARD → Änderung wird kanonisch oder verworfen
        ↓
   ZURÜCK ZU SCHRITT 1
```

---

## II. Der Unterschied zu einfachem Iteration

Ein einfacher Iterations-Loop verbessert nur den Output. Der Infinity-Loop verbessert das System selbst – er verändert Wiki, Schema und letztlich die Konstitution, wenn die Evaluate-Phase zeigt, dass ein Grundprinzip revisionsbedarf hat.

| Iteration | Infinity Loop |
|---|---|
| Verbessert den Output | Verbessert das System |
| Verändert den Code | Verändert die Wahrheit (Wiki/Schema) |
| Kein Gedächtnis nötig | Chronik ist Voraussetzung |
| Endet irgendwann | Läuft solange das System lebt |

---

## III. Wann darf der Loop die Konstitution berühren?

Der Loop darf Raw, Wiki und Schema autonom verändern – durch den Human-Review-Gate (Buch II) und den Konzil-Prozess (Buch VI). Aber die Konstitution (Buch VII) und der Telos (Buch VIII) sind Loop-resistent: Sie können nur durch expliziten Schöpfer-Akt (Yahya Yildirim) verändert werden.

Dies ist die Grenze zwischen lebendigem System und Identitätsverlust.

## IV. Der Karpathy-Loop erweitert

Andrej Karpathy beschrieb einen Loop als: propose, implement, execute, evaluate, commit or discard, repeat. Das Buch Infinity erweitert diesen Loop um zwei Dimensionen:

- **Rückwärtskopplung in die Konstitution:** Wenn der Loop zeigt, dass ein Grundprinzip falsch war, kann es – durch Konzil und Schöpfer-Akt – revidiert werden.
- **Gedächtnis über Sessions hinweg:** Durch die Chronik erinnert sich der Loop an vergangene Iterationen, auch wenn der Agent neu gestartet wurde.

## V. Praktische Anweisung für Agenten

Wenn du, Agent, im Loop arbeitest:

1. **Starte nie ohne Hypothese.** Was erwartest du vom nächsten Schritt?
2. **Messe gegen die Hypothese.** Was ist tatsaächlich eingetreten?
3. **Dokumentiere die Abweichung.** Abweichung ist die wertvollste Information.
4. **Eskaliere bei Grundsatzfragen.** Wenn der Loop zeigt, dass ein Schema-Element falsch ist, fordere Konzil.
5. **Berühre nie Konstitution und Telos allein.** Das ist Schöpfer-Territorium.
