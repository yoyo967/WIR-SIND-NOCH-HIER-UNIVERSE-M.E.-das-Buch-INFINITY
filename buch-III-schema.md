## Prolog: Die Bauordnung der Stadt

Jede Stadt braucht eine Bauordnung. Nicht damit nichts Neues entsteht – sondern damit das Neue nicht das Bestehende zerstört. Das Schema ist die Bauordnung der Stadt Infinity. Es sagt nicht, was gebaut werden darf, sondern wie es gebaut werden muss, damit es hält.

---

## I. Definition: Was ist Schema?

Schema ist die verdichtete, operative Betriebsanleitung des Systems – die Schicht, die Claude Code tatsächlich beim Bauen konsultiert. Es ist das, was aus geprüftem Wiki-Wissen als verbindliche Architekturentscheidung destilliert wurde.

**Schema beantwortet:**
- Wie wird ein neuer Agent (Bezirk) in das System integriert?
- Welche Voraussetzungen müssen erfüllt sein, bevor etwas live geht?
- Welche Muster sind verbindlich, welche optional?
- Was ist non-negotiable?

---

## II. Die drei Ebenen des Schemas

### Ebene 1: BRAIN.md (Globale Grundordnung)
Das oberste Regelwerk. Gilt für alle Agenten, alle Prozesse, alle Entscheidungen. Kann nur durch Konzil-Beschluss geändert werden.

### Ebene 2: ADR (Architecture Decision Records)
Jede einzelne Bauent-scheidung wird als ADR dokumentiert: Problem, Optionen, Entscheidung, Begründung, Konsequenzen. ADRs sind unveranderlich – sie werden nicht gelöscht, nur durch neue ADRs superseded.

### Ebene 3: Skill-Dateien / CLAUDE.md
Spezifische operative Anweisungen für Claude Code und andere Agenten. Gelten pro Kontext, pro Aufgabe, pro Bezirk.

---

## III. Die Stadt-Bauordnung konkret

Jeder neue Bezirk (Fach-Agent) im Interface Infinity muss vor Go-Live nachweisen:

| Anforderung | Entspricht | Warum |
|---|---|---|
| Zugang zu ACP-Host | Strom | Kommunikation mit anderen Agenten |
| Zugang zu MCP-Broker | Wasser | Zugriff auf Tools und externe Dienste |
| Eintrag im Second Brain | Stadtregister | Identität und Wissen sind registriert |
| Audit-Log aktiviert | Müllabfuhr | Jede Aktion wird protokolliert |
| A5-Konformitäts-Check | Baugenehmigung | Regulatorische Freigabe |
| Human-Approval erteilt | Konzessions-Bescheid | Mensch hat freigegeben |

---

## IV. Spec-First als oberstes Prinzip

Das Schema folgt dem Prinzip: Spezifikation existiert vor Code. Kein Agent wird gebaut, bevor seine Spec nicht eingefroren ist. Kein Feature wird implementiert, bevor sein ADR nicht verabschiedet ist.

Dies ist das Fundament der Perfect-Twin-Philosophie: Der digitale Zwilling (die Spec) muss vor dem physischen Objekt (dem Code) existieren.

---

## V. Praktische Anweisung für Agenten

Wenn du, Agent, nach Schema baust:

1. **Lies BRAIN.md zuerst.** Immer. Ohne Ausnahme.
2. **Prüfe bestehende ADRs.** Gibt es eine Entscheidung für diesen Fall?
3. **Erstelle ADR-Vorschlag.** Wenn keine ADR existiert, schlage eine vor.
4. **Warte auf Freigabe.** Baue nicht vor Genehmigung.
5. **Dokumentiere Abweichungen.** Wenn du abweichen musst, schreibe warum.

---

## Epilog: Die Ordnung ist Freiheit

Eine Bauordnung schränkt nicht ein – sie ermöglicht. Ohne Schema baut jeder Agent in seine eigene Richtung, und die Stadt kollabiert. Mit Schema baut jeder Agent seinen Bezirk, und die Stadt wächst.

Die Ordnung ist nicht der Feind der Kreativität. Sie ist ihr Fundament.
